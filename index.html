Currently you can add 2 types of fields , plain input (text) or a drop down. The focus was on being able to create as many fields as you wish, and to spread it across pages. The functionality for now is limited to

  1. Adding a text field
  2. Adding a drop down, and specifying options
  3. Combining the above two types of fields in various pages
  4. You can add a condition to restrict values based on what a user has selected. For example, if user selects Karnataka, limit his option for 'language' to Kannada, English.
  5. The run time (end user) does not use any javascript. This is by design, to cater to all browsers and all platforms. The code generates 100% html
  6. A second application lets the user login and fill up all the forms Run the app
  7. A third application simply displays all the user data that has been filled up so far See this app as well
  8. Documentation is limited, I have provided documentation and help in the actual pages
  9. A major limitation right now is that you can't see a condition. (Though you can add one, and it works - tested). no reason really, wanted to put this up asap
  10. Its been tested on Chrome and Firefox. Final application if developed will work on all browsers. The 'design' app will be supported only on Chrome , Opera and possibly latest IE, the end user application is designed to work on even a browser that does not support javascript

What it does not demonstrate, but is part of the design

  1. Styles, colours, backgrounds, etc. The final pages will have divs with specified class names, and these will be saved in a separate file, and before deployment can be edited to perfect the look and feel
  2. The solution will add fields to the database, and finally provide a css file.This css file can be used to customize the html pages (change alignments, fonts, restructure the relative 'divs' in the page. Reposition them.
  3. Other fields like checkboxes, etc can be easily added. The design already allows it
  4. The html needs to be designed a bit, with the sample style sheets.

Other stuff conceptualized, doable, but not yet done.

  1. Configurable html for the page
  2. Conditional fields - meaning certain data needs to be collected, only on certain input
  3. Customizable error messages - We can associate customizable error messages per field
  4. Basic image manipulation. Allow user to fit images to standard sizes
